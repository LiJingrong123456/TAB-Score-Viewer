# 实施文档

## 1. 环境要求

### 1.1 开发环境

- **Node.js**: >= 18.0.0 (推荐 v20+)
- **npm**: >= 9.0.0
- **操作系统**: Windows / Linux / macOS
- **浏览器**: Chrome 90+, Firefox 88+, Edge 90+, Safari 14+

### 1.2 安装步骤

```bash
# 1. 克隆或下载项目
cd "TAB Score Viewer"

# 2. 安装依赖（使用国内镜像源）
npm install --registry=https://registry.npmmirror.com

# 3. 启动开发服务器
npm run dev

# 4. 浏览器访问
# 默认地址: http://localhost:3000
```

### 1.3 生产构建部署

```bash
# 构建
npm run build

# 预览构建结果
npm run preview

# 构建产物位于 dist/ 目录
# 可直接部署到任何静态文件服务器
```

## 2. Docker部署（可选）

### 2.1 Dockerfile

```dockerfile
# 构建阶段
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci --registry=https://registry.npmmirror.com
COPY . .
RUN npm run build

# 生产阶段
FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### 2.2 构建和运行

```bash
# 构建镜像
docker build -t tab-viewer .

# 运行容器
docker run -p 8080:80 tab-viewer

# 访问 http://localhost:8080
```

## 3. 配置说明

### 3.1 开发服务器配置 (`vite.config.ts`)

```typescript
export default defineConfig({
  server: {
    port: 3000,      // 开发服务器端口（可修改为其他端口）
    open: true,      // 自动打开浏览器
  },
})
```

### 3.2 Tailwind自定义配置 (`tailwind.config.js`)

可调整的主题变量：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `primary` | #3B82F6 | 主色调（蓝色） |
| `accent` | #F97316 | 强调色（橙色） |
| `app-bg` | #121212 | 主背景色 |
| `app-surface` | #1E1E2E | 卡片/表面背景色 |
| `zoom.maxZoom` | 5.0 | 最大缩放比例 |
| `zoom.minZoom` | 0.1 | 最小缩放比例 |

### 3.3 播放参数配置

在 `stores/playerStore.ts` 中修改默认值：

```typescript
const defaultPlayerState: PlayerState = {
  playbackRate: 1.0,   // 默认播放速度 (0.25 - 4.0)
  volume: 0.8,         // 默认音量 (0 - 1)
  loopMode: 'none',    // 默认循环模式: 'none' | 'all' | 'region'
};
```

## 4. 使用指南

### 4.1 打开文件

**方法一：点击上传**
1. 点击「打开」按钮或页面中央的上传区域
2. 选择支持的文件格式
3. 等待加载完成

**方法二：拖拽上传**
1. 直接将文件拖拽到上传区域
2. 松开鼠标完成上传

### 4.2 图片/PDF模式操作

#### 缩放和平移
- **滚轮**: 缩放（以鼠标位置为中心）
- **左键拖动**: 平移画面
- **双击**: 重置视图到初始状态

#### 添加标注
1. 切换右侧面板到「标注」标签
2. 点击谱面任意位置即可添加标注
3. 输入标注文字后按Enter确认
4. 右键标注可更改颜色或删除

#### 使用速度曲线
1. 切换右侧面板到「速度」标签
2. 打开启用开关
3. 选择预设或手动添加控制点
4. 点击图表上的点可精确编辑速度值

### 4.3 GTP模式操作

- GTP文件由AlphaTab自动解析和渲染
- 使用底部播放栏控制播放
- 速度调节直接作用于MIDI播放器
- 标注和速度曲线功能暂不适用于GTP模式

### 4.4 播放控制

| 操作 | 方法 |
|------|------|
| 播放/暂停 | 点击播放按钮 或 按空格键 |
| 跳转进度 | 拖拽进度条 |
| 快进5秒 | 点击 >> 按钮 |
| 快退5秒 | 点击 << 按钮 |
| 调节速度 | 点击 +/- 按钮 或 Ctrl+方向键 |
| 切换循环 | 点击循环图标（无→全部→区域） |

### 4.5 导出标注

1. 在标注面板中点击「导出」按钮
2. 自动下载JSON文件
3. 后续可通过「导入」恢复标注

## 5. 故障排除

### 5.1 常见问题

**Q: PDF无法显示？**
A: 确保网络正常（PDF.js worker从CDN加载）。检查浏览器控制台是否有CORS错误。

**Q: GTP文件打不开？**
A: 目前支持GP3-GP7格式。确保文件未损坏。检查控制台错误信息。

**Q: 播放卡顿？**
A: 图片过大可能导致性能问题。建议压缩图片至2000px宽度以内。

**Q: 标注保存后丢失？**
A: 当前标注存储在内存中。请使用导出功能保存为JSON文件。

### 5.2 性能优化建议

- 图片分辨率建议不超过 2000px 宽度
- PDF页数过多时考虑拆分文件
- 标注数量建议控制在50个以内
- 使用Chrome/Edge浏览器获得最佳性能
