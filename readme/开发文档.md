# 开发文档

## 1. 架构设计

### 1.1 整体架构

```
┌─────────────────────────────────────────────────────┐
│                    App.tsx                          │
│                 (主应用组件)                         │
├──────────┬──────────────────────┬───────────────────┤
│ Toolbar  │   ViewerContainer    │  ControlPanel     │
│ (工具栏)  │   (查看器容器)        │  (控制面板)       │
│          ├──────────┬───────────┤                   │
│          │ ImageView│ PdfViewer │ GtpViewer         │
│          │   er     │           │                   │
├──────────┴──────────┴───────────┴───────────────────┤
│                    PlayBar                           │
│               (播放控制栏)                            │
└─────────────────────────────────────────────────────┘
                        ↕
              ┌─────────────────┐
              │ playerStore     │
              │ (Zustand Store) │
              └─────────────────┘
```

### 1.2 数据流

```
用户操作 → 组件事件 → Store Action → State Update → 组件重渲染
                                    ↓
                              usePlayer Hook
                              (播放逻辑计算)
```

### 1.3 核心设计模式

| 模式 | 应用场景 |
|------|----------|
| **观察者模式** | Zustand Store状态订阅 |
| **策略模式** | 不同格式查看器的自动选择 |
| **工厂模式** | 文件解析器根据扩展名选择解析策略 |
| **单例模式** | AlphaTab API实例管理 |

## 2. 模块说明

### 2.1 类型定义 (`types/index.ts`)

定义了所有TypeScript接口：

```typescript
// 主要类型
ScoreFile        // 文件元信息
ScoreData        // 联合类型: ImageScoreData | PdfScoreData | GtpScoreData  
Annotation       // 标注数据
PlayerState      // 播放器状态
SpeedCurveConfig // 速度曲线配置
```

### 2.2 状态管理 (`stores/playerStore.ts`)

使用Zustand创建的全局Store，包含：

**状态分类：**
- **文件状态**: currentFile, scoreData
- **播放状态**: playState, currentTime, duration, playbackRate, volume, loopMode
- **速度曲线**: speedCurve (points, interpolation, enabled)
- **标注系统**: annotations[]
- **视图状态**: zoom, viewMode, showSidebar

**Actions示例：**
```typescript
setScoreFile(file, data)    // 设置当前文件
togglePlayPause()            // 切换播放/暂停
addAnnotation(annotation)    // 添加标注
applyPreset(preset)          // 应用速度预设
```

### 2.3 Hooks说明

#### usePlayer Hook
- **职责**: 管理基于requestAnimationFrame的播放定时器
- **核心功能**:
  - 速度曲线映射到实际播放速度
  - 循环模式处理（全局/区域）
  - 平滑的时间更新
- **关键函数**: `getSpeedAtTime()`, `tick()`, `seekTo()`

#### useAnnotation Hook
- **职责**: 标注CRUD操作
- **核心功能**:
  - 创建/更新/删除标注
  - 导入/导出JSON
  - 编辑状态管理

#### useSpeedCurve Hook
- **职责**: 速度曲线管理
- **核心功能**:
  - 预设应用
  - 控制点增删改
  - 插值计算（线性/贝塞尔/阶梯）

## 3. 关键算法

### 3.1 速度曲线插值算法

```typescript
// 线性插值
speed = p1.speed + (p2.speed - p1.speed) * t

// 贝塞尔插值 (smoothstep)
t = t * t * (3 - 2 * t)
speed = p1.speed + (p2.speed - p1.speed) * t
```

### 3.2 播放定时器算法

使用`requestAnimationFrame`实现60fps平滑播放：

```typescript
const tick = () => {
  const now = performance.now();
  const elapsed = now - startTime;
  const currentSpeed = getSpeedAtTime(currentTime);
  let newTime = (elapsed * currentSpeed) / 1000;
  
  // 处理循环...
  setCurrentTime(newTime);
  requestAnimationFrame(tick);
};
```

### 3.3 文件格式检测

```typescript
function detectFormat(filename): SupportedFormat {
  const ext = filename.split('.').pop()?.toLowerCase();
  if (IMAGE_EXTENSIONS.includes(ext)) return 'image';
  if (ext === 'pdf') return 'pdf';
  if (GTP_EXTENSIONS.includes(ext)) return 'gtp';
  return 'unknown';
}
```

## 4. 扩展指南

### 4.1 添加新的文件格式支持

1. 在 `types/index.ts` 中添加新类型
2. 在 `FileLoader.tsx` 的 `detectFormat()` 中添加扩展名
3. 创建新的Viewer组件
4. 在 `ViewerContainer.tsx` 中注册

### 4.2 添加新的速度预设

在 `useSpeedCurve.ts` 的 `SPEED_PRESETS` 数组中添加：

```typescript
{
  name: '预设名称',
  description: '描述',
  points: [
    { time: 0, speed: 1.0 },
    { time: 50, speed: 0.8 },
    { time: 100, speed: 1.2 },
  ],
}
```

### 4.3 自定义主题颜色

修改 `tailwind.config.js` 中的颜色映射：

```javascript
colors: {
  'primary': '#YOUR_COLOR',      // 主色调
  'accent': '#YOUR_ACCENT',      // 强调色
  'app-bg': '#YOUR_BG',          // 背景色
  // ...
}
```
