# 功能更新记录

## v1.3.0 (2026-06-07) - Phase 3 音频播放引擎

### 新增功能
- **MIDI事件序列转换器** (`gtp_engine/audio/midi_converter.py`):
  - GTPSong → 带时间戳的 MIDI 事件列表(note_on/note_off/tempo)
  - 基于 tick 时间精度(480 ticks/beat)，避免浮点累积误差
  - 力度处理: 幽灵音→60(弱), 重音→+25(强), 默认95(mf)
  - 断奏(Staccato)自动缩短时值50%
  - 支持 BPM 变化和总时长计算
- **FluidSynth 合成引擎** (`gtp_engine/audio/synth_engine.py`):
  - 基于 python-fluidsynth 开源项目的实时音频合成
  - SoundFont 自动搜索(项目目录/系统路径/用户目录)
  - 独立线程播放，不阻塞 UI 主线程
  - 精确定时(自适应 sleep + busy-wait，误差<5ms)
  - 播放控制: play/pause/stop/seek 完整支持
  - 吉他音色设置(24-30程序号对应各种吉他)
- **音频/视觉同步**:
  - 🔊 音频开关按钮(GTP文件工具栏显示)
  - 当前播放音符高亮显示(顶部颜色条 + 音高文字)
  - 高亮颜色随力度变化(蓝=弱/绿=中/橙=强)
  - 切换音轨自动重建 MIDI 事件
- **优雅降级**: fluidsynth 未安装或 SoundFont 缺失时自动禁用音频按钮并提示

### 新增依赖
- `fluidsynth >= 1.2.0` (python-fluidsynth, 开源项目: FluidSynth)

### 修改文件
- `gtp_engine/__init__.py` — 导出 audio 模块(MidiConverter/SynthEngine)
- `gtp_engine/audio/midi_converter.py` — [新增] MIDI转换器
- `gtp_engine/audio/synth_engine.py` — [新增] FluidSynth合成引擎
- `gtp_engine/audio/__init__.py` — [新增] 包初始化
- `guitar_tab_viewer.py` — 集成音频播放到 DisplayWindow

## v1.2.3 (2026-06-07) - 技巧标签重构 + 击弦弧线 + 滑音s + 调弦简化

### 新增功能
- **击弦/勾弦弧线**: Hammer On / Pull Off 绘制向上凸圆弧连接两个同弦音符，弧线上方显示 H/P 文字
- **滑音s在斜杠上**: 上滑音显示 "s"、下滑音显示 "S"，文字位于斜杠中点
- **技巧标签移至弦线上方**: P.M./let ring/N.H./A.H./T.H./P.H. 全部使用统一虚线算法，画在每行谱子(y_tab_top)上方
- **泛音虚线支持**: N.H./A.H./T.H./P.H. 使用与P.M.相同的跨拍连线虚线算法（橙色 #F59E0B）
- **调弦简化显示**: 使用 `track.get_tuning_name()` 显示"标准调弦(EADGBE)"等，不再显示原始MIDI音高
- **第一页留白**: 第1页第一行系统下移15px，给标题区域更多呼吸空间

### Bug 修复
- **推弦不再显示"B"文字**: BEND已由_draw_bend_indicator图形化处理(弧线+度数文字)，SKIP_TECHS中排除避免重复
- **滑音/击弦不再重复文字**: SLIDE_UP/DOWN/HAMMER_ON/PULL_OFF 均从文字标签中排除

## v1.2.2 (2026-06-07) - 推弦GP5格式渲染 + 休止符规则优化

### Bug 修复
- **推弦箭头必须超出顶线**: 弧线峰值(箭头尖端)现在确保在六线谱最顶线(y_tab_top)之上至少4px。弧线高度动态计算：取基础高度和"起点到顶线外4px距离"中的较大值
- **□ 小方块问题**: 全休止符/二分休止符使用填充矩形 `drawRect` + `setBrush`，在 Solo 轨空小节中显示为白色小方块。改为 `Qt.NoBrush` 细线框样式
- **空小节休止符过多**: Intro 等空小节（无任何音符）不应显示任何休止符符号。新增规则1：检测整个小节是否完全为空（所有拍都是休止），是则完全不画
- **Solo段休止符杂乱**: 同上，空小节过滤后大幅改善

### 新增: GP5格式推弦渲染
推弦标记从简单弧线+文字"B"重写为严格参照 Guitar Pro 5 的标准格式:
- **度数文字**: 显示 "1/4"/"1/2"/"3/4"/"Full"（根据推弦量自动映射）
- **上弯弧线**: 从品格数字下方开始，向上弯曲到峰值，峰值处有向上箭头(▲)
- **释放段**: 有释放的推弦绘制下弯弧线回到原位，末端有向下箭头(▼)
- **动态高度**: 推弦量越大弧线越高（1/4=14px, 1/2=18px, Full=26px）

### 数据模型扩展
- **BendData 类** (`note.py`): 新增推弦详细数据模型，存储类型/度数/曲线点
- **解析器增强** (`gtp_parser.py`): 提取 GTP 库 BendEffect 的完整数据（type/value/points/maxValue）

### 新增: 动态弦数支持
弦线数量不再固定为6根，从GTP文件读取实际弦数:
- **SystemLayout 新增 `string_count` 字段**: 从 `track.string_count` 传入
- **布局引擎**: `tab_height = (string_count - 1) × TAB_LINE_SPACING`
- **渲染器**: `_draw_tab_lines()` 根据 `system.string_count` 绘制对应数量的弦线
- **信息栏**: TAB竖排文字、谱号竖线、拍号位置均根据实际弦数动态调整
- 支持: 4弦(贝斯)、5弦(5弦贝斯)、6弦(标准吉他)、7弦(7弦吉他)等

## v1.2.1 (2026-06-07) - Phase 2 Bug修复与优化

### Bug 修复
- **drawArc float参数崩溃**: 休止符圆弧绘制中 `y_center` 为浮点数导致 PyQt5 类型错误，Solo Guitar 音轨无法渲染。所有坐标参数统一转为 int
- **P.M./Let Ring 虚线格式重写（严格参照 GP5）**:
  - P.M.: 大写 `P.M.` + 跨拍虚线连接 + 末尾停止竖线 `|`
  - Let Ring: 小写 `let ring` + 跨拍虚线 + 停止竖线 `|`（严格模式：基于标准拍间距绝对阈值断开，中间隔1个非LR拍即分段）
  - 虚线长度严格基于 BeatLayout.x_start→x_end（反映每个拍的时值宽度）
  - 连续技巧拍自动分组连线，非连续的独立显示
- **P.M.重复显示**: P.M./Let Ring 同时被 `_draw_technique_text_labels`(每个音符画文字) 和 `_draw_pm_letring_extensions`(虚线+标签) 各画一次。现在文字标签方法跳过 P.M./Let Ring，仅由虚线方法统一绘制
- **休止符每小节末尾都渲染**: GTP文件末尾填充空拍被错误渲染为可见休止符折线。增加过滤：小节最后一个拍且时值≤四分音符时不绘制（判定为填充空拍）
- **四分休止符折线过大**: 锯齿形从六线谱顶部到底部(~50px)，改为紧凑形状限制在中心区域20px高度内
- **信息栏太窄**: TAB+谱号+拍号区域从42px加宽至56px，布局更舒适

### 新增: 独立信息栏（参照 Guitar Pro）
每行系统开头绘制独立信息栏区域（替代原来的简单拍号/调号文字）:
- **T A B 竖排文字**: 最左侧标识六线谱
- **谱号简化符号**: 竖线 + 底部弯钩（模拟高音谱号轮廓）
- **拍号**: 分子/分母上下堆叠（始终显示，GP风格）
- **调号**: 拍号上方灰色小字显示升降号数(#N/bN)
- **双分隔线**: 信息栏右侧双细线与乐谱内容分隔

### 修改文件
- `gtp_engine/renderer/tab_renderer.py` — 重构 `_draw_dashed_extension_line`(时值分段), `_draw_beat`(休止符过滤), `_draw_rest_symbol`(紧凑化+int修复), `_draw_info_bar`(新增,替代旧time/key sig), `_draw_technique_text_labels`(跳过PM/LR)

## v1.2.0 (2026-06-07) - Phase 2 增强渲染

### 新增功能

#### 2.1 技巧符号图形化渲染
- **泛音菱形标记**: 自然/人工/拍击/点弦泛音在品格数字外围绘制菱形框（8px大小），视觉区分明显
- **滑音连线**: 滑音(SLIDE_UP/DOWN)自动查找同弦下一个音符并绘制斜线连接，支持跨拍连线
- **推弦弧线箭头**: 推弦(BEND)技巧绘制向上凸起的贝塞尔曲线弧线 + 箭头尖端，弧高18px可调
- **颤音波浪线**: 颤音(VIBRATO)在品格数字上方绘制正弦波浪线(~~~)，振幅2px
- **P.M./Let Ring 虚线延长**: 手制音/延音技巧绘制虚线从起始音符延伸到结束音符，跨多个拍自动连接
- **技巧文字标签**: 所有非图形化技巧仍显示缩写文字（H/P/s/B等），位于品格数字右侧

#### 2.2 时值显示增强
- **附点标记**: 附点音符(is_dotted)在符尾旁绘制4px圆点标记；休止符附点同样支持
- **休止符符号优化**（替代原来的简单圆点）:
  - 全休止符: 悬挂式矩形块(8×6px)在六线谱上方
  - 二分休止符: 坐落式矩形块在六线谱顶部区域
  - 四分休止符: 锯齿形/闪电形折线符号
  - 八分及更短: 类似"7"的圆弧+竖线组合
- **符尾旗美化**: 符尾旗加粗至1.5px、长度增至7px，十六分/三十二分多符尾平行排列更清晰

#### 2.3 调号/拍号/速度标记
- **拍号显示**: 非标准拍号(非4/4)时在每行系统开头显示上下堆叠的分子分母数字（如3/4、6/8）
- **调号显示**: 非C大调/a小调时在拍号左侧显示升降号数量标注（如#1=1个升号, b2=2个降号）
- **兼容处理**: key_signature同时支持int和tuple两种数据格式

### 修改文件
- `gtp_engine/renderer/tab_renderer.py` — 新增7个方法: `_draw_harmonic_diamond`, `_draw_slide_line`, `_draw_bend_indicator`, `_draw_vibrato_wave`, `_draw_pm_letring_extensions`, `_draw_time_signature`, `_draw_key_signature`; 重构 `_draw_technique_marks`, `_draw_beam_flags`, `_draw_rest_symbol`, `_draw_system`, `_draw_beat`, `_draw_measure`
