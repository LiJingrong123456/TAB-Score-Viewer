# 功能更新记录

## (2026-06-28) - 修复推弦后声音持续尖锐问题 (v1.1.1)

### 修改内容

修复用户反馈的"推完弦后，声音会变得尖锐，一直持续"问题。根本原因是 MIDI 事件排序把 `pitch_bend` reset 事件排到了 `note_off` 之前，且普通音符 `note_on` 前没有防御性复位，导致高位弯音状态泄漏到后续音符。

**修复点 1: 复位事件延迟到 note_off 后**
- `midi_converter.py` `_convert_measure()` 中，`pitch_bend` reset 时间从 `beat_tick + actual_duration` 改为 `+ 1`
- 确保 reset 事件严格在 `note_off` 之后处理，不受排序影响

**修复点 2: 所有音符 note_on 前发送初始 pitch_bend**
- 每个音符开始前都强制设置一次期望的弯音值
- 推弦音符: 发送首点值; 滑入音符: 发送滑入起始值; 普通/揉弦音符: 发送 8192 复位
- 防御性避免通道弯音状态泄漏

**修复点 3: 排序改为稳定排序保留生成顺序**
- `convert()` / `convert_all_tracks()` 末尾排序从"按 type 优先级"改为"仅按 time 稳定排序"
- 生成顺序本身已保证控制/音色事件在音符前，`note_off` 在 reset 前

### 核心原理

1. **MIDI 事件顺序**: 同一 tick 下事件的处理顺序决定最终效果。旧版按 type 排序把 `pitch_bend` 排在 `note_off` 前，导致复位在音符关闭前生效，后续音符继承高位弯音。
2. **通道状态隔离**: 每个音符 `note_on` 前都显式设置一次 `pitch_bend`，即使前一个音符的 reset 因某种原因失效，也能保证当前音符从正确音高开始。
3. **最小偏移 +1 tick**: 既保证 reset 严格在 `note_off` 之后，又不影响播放时序（1 tick ≈ 2ms @120BPM/480tpq，人耳不可感知）。

| `ApolloTab/audio/midi_converter.py` | `_convert_measure()`: 所有音符 note_on 前发 initial pitch_bend; reset 改为 `note_off + 1 tick`; `convert()`/`convert_all_tracks()` 排序改为稳定排序 |
| `_test_bend.py` | 新增集成事件序列检查，验证 reset 在 note_off 后、普通音符前有防御性复位 |
| `readme/功能更新.md` | 新增本次更新记录 |

## (2026-06-28) - 修复 GTP 播放音色发闷问题 (v1.1.1)

### 修改内容

修复用户反馈的"播放时好多音都有点闷，但开头正常"问题。根本原因是 MIDI 输出没有正确应用谱面指定的乐器音色，导致部分音轨错误使用了钢琴(program=0)或被播放器强制覆盖为单一清音电吉他(program=27)。

**修复点 1: midi_converter.py 默认音色**
- 旋律轨 `track.instrument == 0` 时，不再发送 `Program Change 0`(大钢琴)
- 改为发送 `Program Change 25`(钢弦吉他，Acoustic Guitar Steel)
- GP 文件明确指定音色时(非0)，仍按原值发送
- 新增类常量 `DEFAULT_MELODY_INSTRUMENT = 25`，便于后续统一调整

**修复点 2: player.py 移除强制音色覆盖**
- 删除 `rebuild_audio_events()` 中"为每个通道设置合适的乐器音色"的循环
- 不再调用 `set_instrument(ch, 27)` 把所有非鼓通道强制设为清音电吉他
- 音色完全由 `MidiConverter` 生成的 `Bank Select + Program Change` 事件决定
- 保留 `convert_all_tracks` 原有音色事件生成逻辑，GP7/GP8 解析的 bank 和 program 可正确生效

### 核心原理

1. **GM 音色号**: 0=Grand Piano(钢琴，延音长、听起来闷), 25=Acoustic Guitar Steel(钢弦吉他)
2. **事件时序**: `Bank Select MSB(CC#0) → Bank Select LSB(CC#32) → Program Change` 仍保持原有排序，确保 bank/program 正确切换
3. **播放状态一致性**: 移除 player 层的强制覆盖后，`seek()` 到中后段时不会再因跳过 `tick=0` 的 program_change 而错用 27 号音色

### 修改文件

| 文件 | 改动 |
| --- | --- |
| `ApolloTab/audio/midi_converter.py` | 新增 `DEFAULT_MELODY_INSTRUMENT=25`；`_create_program_events` 中 instrument=0 时默认使用 25 号钢弦吉他 |
| `ApolloTab/player.py` | 删除 `rebuild_audio_events()` 中强制 `set_instrument(ch, 27)` 的循环，音色由 midi_converter 统一处理 |
| `readme/功能更新.md` | 新增本次更新记录，按规则移除最旧的"设置面板: 添加恢复默认功能"记录以控制文件长度 |

## (2026-06-28) - 推弦/滑弦/揉弦 MIDI 合成对齐 alphaTab (v1.1.0)

### 修改内容

将 ApolloTab 的推弦/滑弦/揉弦 MIDI 合成逻辑对齐 `alphaTab-develop` 的 `MidiFileGenerator.ts` 实现（参考 `readme/alphaTab推弦MIDI合成原理分析.md`）。本次只实现 Bend/Slide/Vibrato 三种效果，**不**包含 Whammy（摇把）。

### 核心原理

1. **RPN 设置弯音范围**：每个旋律音轨开头发送 `CC#101=0 → CC#100=0 → CC#6=12 → CC#38=0`，使合成器 Pitch Bend Range = ±12 半音。
2. **Pitch Wheel 公式**：`8192 + (bend_value / 2) * (8192 / 12)`，`bend_value` 单位为 1/4 半音（4 = Full = 2 半音）。
3. **双重密度插值**（`_generate_bend_values`）：在相邻 BendPoint 间按 `max(每半音×6断点, 时长ms/150)` 插值，保证曲线平滑且事件量受控。
4. **技巧优先级互斥**：bend > slide > vibrato（与 alphaTab 一致），同一音符只生成一种效果。
5. **通道隔离防泄漏**：note_on 前发送初始弯音值，note_off 后强制复位 8192。
6. **滑弦前瞻**：构建扁平音符列表 `flat_notes`，`_compute_slide_dy` 按位置向后查找同弦下一音计算 dy。
7. **揉弦正弦波**：按 `VIBRATO_RESOLUTION_TICKS=16` 步长生成密集 pitch_bend 事件，Slight 振幅 1.5 / Wide 振幅 3.0（1/4 半音单位），周期 320ms。

### 修改文件

| 文件 | 修改要点 |
|------|---------|
| `ApolloTab/utils/constants.py` | 新增 `BendType`（9 值）、`BendStyle`（3 值）、`VibratoType`（3 值）枚举 |
| `ApolloTab/models/note.py` | `BendData` 改用枚举字段，单位统一为 1/4 半音；`get_display_text` 映射改为 `{1:"1/4", 2:"1/2", 3:"3/4", 4:"Full"}`；`GTPNote` 新增 `vibrato` 字段 |
| `ApolloTab/parser/gtp_parser.py` | GP3-5：BendPoint position ×5 转 0-60，value 不变（已是 1/4 半音）；bend_type 用 `_GP_BEND_TYPE_MAP` 映射；摇把类型映射为 `CUSTOM`；vibrato 写入 `VibratoType.SLIGHT` |

### 10 条测试用例及预期结果

| # | 输入 | 预期结果 |
|---|------|---------|
| 1 | `get_pitch_wheel(0)` | 8192（原位） |
| 2 | `get_pitch_wheel(4)` | 9557（Full bend = +2 半音） |
| 3 | `get_pitch_wheel(-4)` | 6827（-Full = -2 半音） |
| 4 | `_create_pitch_bend_range_events(0)` | 4 个 CC 事件：101=0, 100=0, 6=12, 38=0 |
| 5 | Full bend 音符（4/4 拍）| 生成 12 个 pitch_bend，首值 8192，末值 9557 |
| 6 | Slight 揉弦音符（2 拍）| 生成 64 个 pitch_bend，值围绕 8192 波动（7684~8703） |

| 10 | GP3-5 文件中 position=12 的点 | 转换为 60（×5），表示音符末尾 |

### 单位一致性

- **GP3-5**：PyGuitarPro 的 `position` 0-12 经 `×5` 转为 0-60；`value` 已是 1/4 半音单位，无需转换。
- **GP7/GP8**：GPIF XML 的 `offset` 0-100 经 `×0.6` 转为 0-60；`value` 已是 1/4 半音单位（25→1, 100→4）。
- **渲染器**：`base_arc_map` 的 key 改为 `{1,2,3,4}`，与 `bend.max_value` 一致。
