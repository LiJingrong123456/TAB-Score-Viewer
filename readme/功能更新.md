# 功能更新记录

## v2.0.7 (2026-06-15) - 异步导出+JPG格式支持

### 新功能

- **异步导出系统 (解决大文件UI冻结)**:
  - 导出操作改为后台线程(QRunnable + QThreadPool)执行，UI不再冻结
  - 新增 `ExportProgressDialog` 进度对话框:
    - 实时显示渲染状态(收集数据 / 渲染轨道X/Y / 保存文件 / 完成)
    - 进度条可视化展示整体进度
    - 取消按钮可随时中断导出
    - 完成/出错/取消后自动关闭并提示结果
  - 新增 `ExportWorker` 工作线程类: 通过信号报告进度/完成/错误/取消

- **JPG格式导出支持**:
  - ExportDialog格式选项新增 **JPG (有损压缩,适合分享)**
  - 新增JPG质量滑块(1-100，默认90)，仅选择JPG时显示
  - 质量参数说明: 90=高质量接近无损 / 80=网络分享 / 60=极致压缩
  - 文件体积比PNG小3-5倍，适合微信/QQ/论坛分享
  - 新增 `_render_to_a4_jpg()` 方法: 基于PNG渲染逻辑，使用Qt的JPEG质量参数输出

### 架构变更

- **导出流程从同步改为异步**:
  - 旧版(v2): `_export_to_a4()` → UI主线程直接调用渲染 → 大文件冻结5-10秒
  - 新版(v3): `_export_to_a4()` → 创建ExportWorker → QThreadPool后台运行 → 信号驱动进度更新
  - ExportWorker持有DisplayWindow引用，调用其渲染方法完成实际工作
  - ExportProgressDialog通过信号槽机制接收进度更新(跨线程安全)

### 技术实现

- **ExportWorkerSignals**: 自定义信号类(progress/finished/error/cancelled)，继承QObject支持pyqtSignal
- **ExportWorker(QRunnable)**: 
  - run()方法中依次执行: 收集数据 → 逐轨渲染(每轨前检查取消标志) → 报告完成
  - cancel()方法设置原子标志位，下个渲染循环生效
- **ExportProgressDialog(QDialog)**:
  - set_worker()绑定ExportWorker实例和信号连接
  - 进度条计算: 每轨道占(100/总轨道)分，当前轨道内按页码细分
  - 自动关闭: 完成500ms后 / 出错500ms后 / 取消800ms后
- **_render_to_a4_jpg()**: 
  - 复用PNG的预渲染防溢出方案(page_buffer → A4画布)
  - QImage格式用RGB32(无alpha通道，比ARGB32更省空间)
  - Qt质量映射: 用户1-100 → Qt内部0-99

## v2.0.6 (2026-06-14) - GTP音轨音量控制(专业DAW混音器风格)

### 增量修复

| # | 问题 | 根因 | 修复 |
|---|------|------|------|
| 1 | **A/B循环到达B点后不seek回A点** | UI层`_tick()`在`effective_time >= total_dur`时提前调用`stop_playback()`，导致库层来不及触发循环重启 | region循环模式下跳过UI层播放结束检测，交给库层管理循环生命周期 |
| 2 | **点击谱面跳转到小节中间位置** | 旧方案使用精确像素定位，可能跳到小节内部任意位置，与A/B循环的小节原子单位不一致 | 重构为以"小节"为单位定位: 点击任意位置自动对齐到所在小节的起始拍 |
| 3 | **A/B循环激活时点击区间外小节会跳出循环** | 点击跳转未检查A/B循环范围，任何位置都可跳转 | 循环激活时判断目标小节是否在[loop_start, loop_end]区间内，区间外则忽略跳转 |
| 4 | **每行(System)小节索引重复导致跨行定位错误** | `build_timeline()`中`meas_idx`通过`enumerate(system.measures)`获取，每个系统(行)内从0重新计数，导致不同系统的同名小节(如都是小节0)被合并 → 循环设置/点击跳转选错小节 | 引入全局唯一ID `global_meas_idx`(递增计数器)，保留原`meas_idx`用于UI显示；所有内部匹配使用`global_meas_idx`确保跨系统/页唯一性 |
| 5 | **点击空白小节(无音符)误差大** | 空白小节在`build_timeline()`中被`continue`跳过，不生成timeline条目 → 点击时二分查找落到相邻有音符的小节 | 为空白小节生成单个占位条目(`beat_idx=-1`)，记录其起始scroll_y和time_ms，使点击可正确定位 |

### 新功能

- **小节单位点击跳转(与A/B循环联动)**:
  - GTP模式: 点击谱面任意位置 → 自动对齐到所在小节的起始拍并开始播放
  - A/B循环联动:
    - 循环激活 + 点击区间外小节 → 忽略跳转，维持循环播放
    - 循环激活 + 点击区间内小节 → 跳转到该小节开头并继续循环
    - 循环未激活 → 正常跳转到点击的小节开头
  - 非GTP模式(图片/PDF): 降级为原始像素级跳转(无时间线数据)
  - 边界容差±50ms: 避免边界值因时间精度被误判为"区间外"

- **GTPPlayer新增API**:
  - `find_measure_at_scroll_pos(scroll_y)`: 根据滚动Y位置查找所在小节信息(返回meas_idx/start_time_ms/start_scroll_y)
  - `loop_time_range`属性: 获取当前A/B循环的时间范围(毫秒元组)

- **GTP音轨音量调节滑块**:

- **TrackVolumeSlider组件**:
  - 自定义垂直音量滑块QWidget(继承QObject支持pyqtSignal)
  - dB值与像素坐标双向转换(_db_to_y / _y_to_db)
  - 渐变填充轨道 + 3D立体手柄效果
  - 动态主题适配(深色/浅色模式自动切换配色)
  - 信号机制: valueChanged.emit(db_value) 实时通知音量变化

### 技术实现

- **UI布局**: QGroupBox → QHBoxLayout → QScrollArea → QWidget(容器) → 多个TrackVolumeSlider
- **动态创建**: _setup_volume_controls() 根据GTP文件音轨数量动态生成滑块
- **数据存储**: _track_volumes: Dict[int, float] 存储各轨音量, _master_volume存储总音量
- **音频引擎集成**: 尝试调用 GTPPlayer.set_track_volume() / set_master_volume() (如果底层支持)
- **可见性控制**: 仅GTP文件显示，非GTP文件/关闭时自动隐藏

## v2.0.4 (2026-06-14) - A/B循环架构重构(下沉到库层，基于小节原子单位)

### 架构变更

- **A/B循环逻辑从UI层完全移除，下沉到库层(SynthEngine音频线程内部)**:
  - 用户核心思路: "把每个小节都作为一个模块，A在哪个小节就从那个小节开始播，B在哪个小节就在那个小节末尾结束"
  - **旧方案的问题**(v2.0.3及之前): UI层用帧计数器/冷却/模拟时钟/安全边界等复杂机制试图解决竞态→每修一个bug引出新的→无限循环
  - **新方案的原理**:
    1. `SynthEngine._play_loop()` 内置循环: 播放完所有事件后检查`_loop_enabled`标志 → 若启用则静音→重置时间基准→重新遍历事件列表（早于loop\_start的事件被time检查自动跳过）
    2. `GTPPlayer` 提供基于小节的API: `set_loop_region_by_position(pct, pct)` 将百分比→转换为小节索引→对齐到小节边界(起始小节第一拍/结束小节最后一拍)
    3. UI层 `_tick()` 恢复为干净的"读audio\_time\_ms→算position"，零循环相关代码
  - **彻底消除的问题**: 竞态条件、冷却期、模拟时钟、安全边界、停顿、跳动、卡死 — 全部不存在了
