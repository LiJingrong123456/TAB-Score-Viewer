# 功能更新记录

## v2.0.0 (2026-06-13) - 深色/浅色主题系统 + UI美化

### 新功能
- **ThemeManager 主题管理器**: 全局深色/浅色主题统一管理
  - **单例模式**: ThemeManager 类全局唯一实例，管理两套完整配色方案
  - **动态切换**: 运行时实时切换深色/浅色主题，无需重启应用
  - **信号通知**: theme_changed 信号发射，所有UI组件自动刷新样式
  - **GTP渲染联动**: UI主题与ApolloTab渲染主题自动同步
    - 浅色UI → GTP谱使用 light 渲染主题
    - 深色UI → GTP谱使用 dark 渲染主题（可手动切换）
    - 导出GTP谱 → 强制使用 light 浅色渲染（适合打印）

- **浅色主题配色方案(THEME_LIGHT)**: 精心设计的浅色模式配色
  - 背景色: #F8FAFC(主背景) / #FFFFFF(表面)
  - 文字色: #1E293B(主文字) / #64748B(次要文字)
  - 主色调: #3B82F6(蓝色) / #60A5FA(浅蓝)
  - 成功/警告/错误: #10B981 / #F59E0B / #EF4444
  - 边框/圆角: 统一8px圆角 + 1px边框

- **自定义组件主题适配**:
  - ModernButton: 支持 primary/success/warning/danger/accent 5种语义色
  - ModernSlider: 支持多色系滑块(primary/accent/text_muted)
  - ProgressBarSlider: 进度条颜色随主题动态变化
  - 所有组件从 ThemeManager.get() 获取当前主题色

- **主题切换UI控件**:
  - 工具栏按钮: 🌙/☀️ 图标按钮，点击即时切换主题
  - 设置窗口下拉框: "深色模式" / "浅色模式" 选择
  - 按钮美化: 圆角6px、半透明背景、悬停高亮效果

- **主题持久化**:
  - 用户选择保存到 settings.json 的 `theme` 字段
  - 启动时自动恢复上次使用的主题
  - 默认主题: dark（深色模式）

- **国际化支持**:
  - 新增翻译key: toolbar.theme_tooltip, settings_window.theme_label
  - 中文/英文翻译文件已同步更新

### 技术原理
- **设计模式**: 单例模式(Singleton) + 观察者模式(Observer/Signal)
- **配色字典**: THEME_DARK / THEME_LIGHT 两套独立配色，通过 ThemeManager 统一访问
- **向后兼容**: 保留全局变量 THEME_COLORS = ThemeManager._theme_data
- **渲染主题映射**: get_gtp_render_theme() 方法返回 ApolloTab 需要的主题名

### 修改文件
- **重构** `TAB Score Viewer.py` — 新增ThemeManager + THEEM_LIGHT + _refresh_theme() + 主题UI + GTP渲染联动
- **更新** `locales/zh_CN.json` — 新增 theme_label, theme_tooltip 翻译
- **更新** `locales/en_US.json` — 新增 theme_label, theme_tooltip 翻译

## v1.9.1 (2026-06-12) - 应用图标 + EXE打包支持

### 新功能
- **应用图标系统**: icon.ico 窗口图标 + 任务栏图标统一显示
  - **多尺寸ICO**: 从 icon.png (2048x2048) 转换生成含 16/32/48/64/128/256px 的 ICO 文件
  - **全局应用图标**: `QApplication.setWindowIcon()` 设置任务栏图标和默认窗口图标
  - **窗口图标**: DisplayWindow 和 SettingsWindow 均调用 `setWindowIcon()` 设置独立窗口图标
  - **路径兼容**: `get_app_icon()` 函数自动适配开发模式(脚本目录)和打包模式(exe目录)

- **PyInstaller 打包配置**: 新增 `guitar_tab_viewer.spec` 非单文件打包配置
  - **onedir模式**: 输出目录(非单文件)，启动更快，无需解压
  - **数据文件包含**: locales/翻译、音频DLL(libfluidsynth-3/SDL3/sndfile)、icon.ico
  - **隐藏导入**: ApolloTab 全模块声明，确保 GTP 功能完整打包
  - **UPX压缩**: 启用体积优化（需安装 UPX）
  - **无控制台**: GUI 应用不显示命令行窗口

### 技术原理
- **图标路径解析**: `sys.frozen` 检测是否为 PyInstaller 打包环境
  - 开发模式: `os.path.dirname(os.path.abspath(__file__))` → 脚本目录
  - 打包模式: `os.path.dirname(sys.executable)` → exe 所在目录
- **ICO多尺寸**: Windows 根据显示场景(DPI/位置)自动选择最合适的尺寸
- **PyInstaller onedir vs onefile**: onedir 将依赖放在 exe 同目录下，避免运行时解压

### 修改文件
- **修改** `guitar_tab_viewer.py` — 新增 get_app_icon() 函数 + 应用/窗口图标设置
- **新增** `guitar_tab_viewer.spec` — PyInstaller 非单文件打包配置
- **新增/更新** `icon.ico` — 多尺寸应用图标文件

## v1.9.0 (2026-06-12) - 多语言支持(i18n)

### 新功能
- **国际化系统(I18n)**: 完整的多语言支持框架
  - **单例模式**: I18n 类全局唯一实例，统一管理翻译状态
  - **JSON翻译文件**: locales/ 目录下按语言存放翻译文件（zh_CN.json / en_US.json）
  - **占位符支持**: 使用 Python str.format() 语法，如 `"位置: {pct}%"`
  - **语言回退机制**: 缺失翻译自动回退到中文(zh_CN)，确保UI不会出现空白
  - **信号通知**: language_changed 信号，连接后可在语言变更时刷新UI
  - **配置持久化**: 语言选择保存到 settings.json 的 `language` 字段

- **语言切换UI**: 设置窗口新增语言选择下拉框
  - 支持中文(简体)和英文两种语言
  - 切换后立即保存配置，新打开的窗口使用新语言
  - 提示用户部分静态文本需重启应用完全生效

- **全面国际化覆盖**:
  - 主窗口(DisplayWindow): 工具栏/控制面板/状态信息/消息提示
  - 设置窗口(SettingsWindow): 文件夹选择/搜索/右键菜单/错误提示
  - 对话框: 标注创建/编辑/管理器、导出对话框、速度曲线编辑器
  - 底部进度栏、音频模式按钮、A-B循环提示等

### 技术原理
- **翻译键命名规范**: 模块.子模块.具体项 (如 control_panel.play_btn)
- **调用方式**: `I18n.t("key", **kwargs)` 获取翻译文本
- **语言检测**: 启动时从 settings.json 读取已保存的语言设置
- **扩展性**: 新增语言只需在 locales/ 目录添加对应的 JSON 文件

### 修改文件
- **重构** `guitar_tab_viewer.py` — 新增I18n类 + 所有硬编码字符串替换为t()调用 + 语言切换UI
- **新增** `locales/zh_CN.json` — 简体中文翻译文件（完整覆盖所有UI文本）
- **新增** `locales/en_US.json` — 英文翻译文件（完整覆盖所有UI文本）

## v1.8.1 (2026-06-12) - 主程序GTP代码库化完成

### 重构内容
- **完全解耦**: 主程序 `guitar_tab_viewer.py` 中所有GTP相关代码已迁移至 `gtp_engine` 库
  - 删除内联的 `_synth_engine`, `_midi_converter`, `_gtp_song`, `_audio_events` 等属性
  - 删除内联的 `_build_playhead_timeline()`, `_update_playhead()`, `_time_to_scroll_pos()`, `_scroll_pos_to_time()` 方法实现
  - 统一使用 `GTPPlayer` 高级封装类（整合解析/渲染/音频/时间线功能）

- **接口统一**: 所有GTP操作通过 `self.gtp_player` 单一入口访问
  - 文件加载: `player.load()` → 自动解析+渲染+准备音频数据
  - 音轨管理: `player.current_track`, `player.get_all_tracks_info()`, `player.render_track()`
  - 播放控制: `player.play()`, `player.stop()`, `player.seek()`, `player.shutdown()`
  - 时间线: `player.build_timeline()`, `player.time_to_scroll_pos()`, `player.scroll_pos_to_time()`
  - 音频模式: `player.init_audio()`, `player.set_audio_mode()`

- **代码清理**: 移除约200行冗余代码，主程序更简洁易维护
  - 播放控制方法（start/closeEvent/stop_playback）改用gtp_player调用
  - 进度条拖动同步简化为单次`player.seek()`调用
  - 标注系统适配：通过`player.current_track`和`player.file_path`获取状态

### 技术原理
- **封装模式**: GTPPlayer作为Facade（外观模式），隐藏内部复杂实现细节
- **延迟初始化**: GTPPlayer在加载GTP文件时创建，非GTP文件时为None（零开销）
- **状态同步**: gtp_player内部维护完整的音频/时间线状态，主程序只做UI展示

### 修改文件
- **重构** `guitar_tab_viewer.py` — 删除所有内联GTP代码，改为调用gtp_player库
- **新增** `gtp_engine/player.py` — GTPPlayer高级播放器封装类（Phase 4已创建）
- **更新** `gtp_engine/__init__.py` — 导出GTPPlayer和相关工厂函数

## v1.8.0 (2026-06-12) - Phase 4: gtp_engine 库化（PyPI发布）

### 新功能
- **PyPI包发布配置**: 新增 `pyproject.toml` 配置文件，支持标准Python包构建和发布
  - 使用现代 setuptools + wheel 构建系统（兼容 Python 3.8+）
  - 完整的元数据定义：名称/版本/描述/作者/分类器/依赖
  - 支持可选依赖组：`full`（完整功能）, `dev`（开发环境）
  - 包含 black/isort/mypy 代码质量工具配置

- **独立包结构优化**: gtp_engine 完全解耦，可作为独立库安装使用
  - 零主程序依赖：无任何对 guitar_tab_viewer.py 的引用
  - 添加 PEP 561 `py.typed` 类型提示标记文件
  - 完善的子模块 `__init__.py` 导出定义
  - 支持 `pip install .` 或 `pip install -e .` 开发模式安装

- **完整文档体系**:
  - `gtp_engine/README.md`: PyPI级README，含快速开始/API参考/配置说明
  - `examples/basic_parse.py`: 基础解析示例（提取歌曲信息/音轨列表）
  - `examples/render_tab.py`: 渲染示例（自定义参数/批量转换）
  - `examples/audio_playback.py`: 音频播放示例（交互控制/进度显示）

- **许可证与分发**:
  - MIT License 许可证文件 (`gtp_engine/LICENSE`)
  - MANIFEST.in 打包配置（包含README/LICENSE/示例代码）

### 技术原理
- **pyproject.toml**: PEP 517/518 标准，替代传统 setup.py/setup.cfg
- **模块解耦**: gtp_engine 仅依赖 pyguitarpro/PyQt5/pyfluidsynth，无项目内部耦合
- **类型支持**: py.typed 文件使 mypy 等工具可检查库的类型注解

### 使用方式
```bash
# 安装（从PyPI，发布后）
pip install gtp-engine

# 或从源码安装
cd "gtp_engine所在目录"
pip install -e .

# 使用示例
from gtp_engine import parse_gtp, render_gtp, SynthEngine
song = parse_gtp("my_song.gp5")
pages = render_gtp("my_song.gp5", track_index=0)
```
