## (2026-07-23) - 修复设置界面主题切换不立即生效 (v2.4.1)

### 问题描述

打开「设置」切换主题后，设置对话框背景变了，但**主窗口（SelectionWindow）和其他区域**仍保持旧主题，需要重启程序才生效。

### 根因分析

`settings_dialog.py` 通过 `importlib.import_module("TAB Score Viewer")` 延迟获取 `SelectionWindow` 类做 `isinstance` 检查。但是当主文件以 `python "TAB Score Viewer.py"` 启动时，Python 将其注册为 `sys.modules['__main__']`，**不是** `'TAB Score Viewer'`。

`importlib.import_module("TAB Score Viewer")` 会**重新创建**一个模块实例，导致返回的 `SelectionWindow` 类对象与 `self._parent_window` 实际所属的类**不是同一个对象** —— `isinstance` 永远返回 `False` → `_refresh_parent_themes` 直接 `pass` 跳过 → 主窗口的 `setStyleSheet` 从未被调用 → 视觉不更新（虽然 `ThemeManager.set_theme` 内部 `theme_changed` 信号仍然触发了 DisplayWindow 刷新，但 SelectionWindow 的 QSS 走的是独立路径）。

### 修复

- `settings_dialog.py::_get_selection_window_cls` 改为优先从 `sys.modules.get('__main__')` 获取 `SelectionWindow` 类，避免 `importlib` 重新加载，保证类身份一致。`importlib` 路径作为兜底保留。
- `settings_dialog.py::__init__` 中给 `SettingsDialog` 设置 `objectName="SettingsDialog"`；`SelectionWindow.__init__` 中设置 `objectName="SelectionWindow"`，为后续可能的精确 QSS 选择器提供锚点（当前未使用，不影响行为）。
- `SettingsDialog` 在 `__init__` 中连接 `ThemeManager.theme_changed → self._on_theme_changed`（防御性信号槽）；`SelectionWindow` 同样连接 `ThemeManager.theme_changed → self._on_theme_changed_external`。两条兜底信号链路在 `_refresh_parent_themes` 之外提供冗余刷新。

### 修改文件

| 文件 | 改动 |
| --- | --- |
| `settings_dialog.py` | `_get_selection_window_cls` 改用 `sys.modules.get('__main__')`；`SettingsDialog.__init__` 添加 `setObjectName` 和 `theme_changed` 信号连接；新增 `_on_theme_changed` 槽 |
| `TAB Score Viewer.py` | `SelectionWindow.__init__` 添加 `setObjectName` 和 `theme_changed` 信号连接；新增 `_on_theme_changed_external` 槽 |

### 验收

| 场景 | 预期结果 |
| --- | --- |
| 打开设置 → 切换主题 | 设置对话框和主窗口**同时**立即变色 |
| 切换深色 ↔ 浅色 | 两个窗口同步切换，无需重启 |
| 切换主题后立即关闭设置 | 主窗口保留新主题，不回退 |
| 多窗口连续切换 | 每次都即时生效 |
| 启动时不打开设置 | 启动时使用上次保存的主题，正常显示 |

## (2026-07-22) - 难度评分 (v2.4.0)

### 新增内容

GTP 文件自动难度评分（0-10 星），帮助用户在打开前快速评估曲目难度。

1. **评分机制**：
   - 仅对 **GTP 类文件**（`.gp` / `.gp3-5` / `.gpx` / `.gtp`）生效
   - **静默后台解析**（`QRunnable` + `QThreadPool`），不阻塞 UI
   - **SQLite 缓存**（`data/difficulty_cache.db`），按文件 mtime 自动失效
   - 解析失败 / 无效音轨 → 不显示星号
2. **评分公式**（0-10）：
   - `base = 2.0`（基础分）
   - `tech_bonus = Σ(tech_count × weight) / (track_count + 1)`
   - `bpm_bonus = min(2.0, bpm / 80)`
   - `dur_bonus = min(1.5, duration_sec / 600)`
   - 最终 `score = clamp(base + tech + bpm + dur, 0, 10)`
3. **音轨过滤**（**仅评吉他家族**）：
   - 排除：鼓（`is_percussion` / 名称含 drum）/ 钢琴 / 键盘 / 鼓机 / 贝斯 / 合成 / 管弦 / 人声
   - MIDI program 范围 0-23 / 32-79 全部排除；24-31（吉他）保留
   - 仅保留：含 `Guitar` / `Acoustic` / `Electric` / `Banjo` / `Mandolin` 等的音轨
   - 所有音轨被过滤 → `score=0`，不显示星号
4. **技术权重**（按用户定义，**较难 1.5+ / 较易 0.4**）：

| 技巧 | 权重 | 难度 |
|------|------|------|
| Bend（推弦） | 1.3 | 较难 |
| Harmonic（泛音） | 1.3 | 较难 |
| Tapping（点弦） | 1.8 | 较难 |
| Vibrato（颤音） | 0.8 | 较难 |
| Whammy（摇杆） | 1.2 | 较难 |
| Trill（击弦颤音） | 1.2 | 较难 |
| Tremolo（震音） | 1.5 | 较难 |
| Slide（滑弦） | 0.3 | 较易 |
| Hammer（击弦） | 0.3 | 较易 |
| Pull（勾弦） | 0.3 | 较易 |

5. **行文本格式**：
   - 评分 ≥ 5.0：`<name>  ★ 6.2/10`
   - 评分 < 5.0：`<name>  ☆ 3.5/10`
   - 非 GTP / 解析失败：不显示
6. **鼠标悬停 Tooltip**（多行明细）：
   ```
   难度评分: ★ 7.3/10
   ───────────────
   有效音轨: 1
   BPM: 120
   时长: 0:06
   ───────────────
   技术统计:
     • Bend (推弦): 0
     • Harmonic (泛音): 5
     ...
   ───────────────
   评分因子:
     base=2.0 tech=+1.7 bpm=+1.5 dur=+0.0
   ```

### 核心原理

- **网络层**：`urllib.request` 同步请求（5 秒超时）→ `check_for_update()` 返回 `UpdateResult` 数据类。`UpdateCheckWorker` 用 `QRunnable` 包装，工作线程完成后通过 `pyqtSignal` 派发回主线程，Qt 信号机制保证线程安全。
- **状态机**：`UpdateStatus` 枚举 4 个状态，UI 层根据状态决定弹窗 / print 静默。
- **QSS 缓存**：`{theme_name: {window_class: qss_string}}` 嵌套字典；切换主题时自动失效；同一主题同一窗口类下多次刷新零开销。
- **图标加载**：开发模式从 `_APP_BASE_DIR` 读，PyInstaller 模式从 `sys.executable` 所在目录 + `_internal/` 子目录读，跨平台三格式支持。

## (2026-07-22) - 项目代码结构重构：拆分核心模块 (v2.2.0)

### 修改内容

为降低 `TAB Score Viewer.py`（已超 6000 行）的维护成本，按职责将核心代码抽离到独立的扁平模块（与主文件同级）。所有抽离的模块文件统一放置在项目根目录，不引入子包结构，便于 PyInstaller 打包。

1. **Phase 0：建立包结构占位**：最初创建 `tab_score_viewer/` 子包存放 `constants.py` 与 `__init__.py`。
2. **Phase 1：常量与配置抽离**：
   - `constants.py`：全局路径（`_APP_BASE_DIR` / `CONFIG_FILE` / `ANNOTATION_DIR` / `CUSTOM_THEMES_DIR` / `ICON_PATH`）、文件扩展名常量、`THEME_DARK` / `THEME_LIGHT` / `THEME_COLORS`、`_RENDER_PARAMS` 元组表、`_DEFAULT_LANGUAGE` / `_DEFAULT_THEME` / `_DEFAULT_GTP_FONT`。
3. **Phase 2：数据模型抽离** (`models.py`)：
   - `Annotation`：标注数据类（id / 坐标 / 文本 / 字号 / 粗体 / 颜色）。
   - `SpeedCurve`：速度曲线相关数据。
   - `LoopConfig`：A-B 循环配置。

## (2026-06-28) - 修复 GTP 播放音色发闷问题 (v1.1.1)

### 修改内容

修复用户反馈的"播放时好多音都有点闷，但开头正常"问题。根本原因是 MIDI 输出没有正确应用谱面指定的乐器音色，导致部分音轨错误使用了钢琴(program=0)或被播放器强制覆盖为单一清音电吉他(program=27)。

**修复点 1: midi_converter.py 默认音色**
- 旋律轨 `track.instrument == 0` 时，不再发送 `Program Change 0`(大钢琴)
- 改为发送 `Program Change 25`(钢弦吉他，Acoustic Guitar Steel)
- GP 文件明确指定音色时(非0)，仍按原值发送
- 新增类常量 `DEFAULT_MELODY_INSTRUMENT = 25`，便于后续统一调整

**修复点 2: player.py 移除强制音色覆盖**
- 删除 `rebuild_audio_events()` 中"为每个通道设置合适的乐器音色"的循环
- 不再调用 `set_instrument(ch, 27)` 把所有非鼓通道强制设为清音电吉他
- 音色完全由 `MidiConverter` 生成的 `Bank Select + Program Change` 事件决定
- 保留 `convert_all_tracks` 原有音色事件生成逻辑，GP7/GP8 解析的 bank 和 program 可正确生效

### 核心原理

1. **GM 音色号**: 0=Grand Piano(钢琴，延音长、听起来闷), 25=Acoustic Guitar Steel(钢弦吉他)
2. **事件时序**: `Bank Select MSB(CC#0) → Bank Select LSB(CC#32) → Program Change` 仍保持原有排序，确保 bank/program 正确切换
3. **播放状态一致性**: 移除 player 层的强制覆盖后，`seek()` 到中后段时不会再因跳过 `tick=0` 的 program_change 而错用 27 号音色
